## 项目概览

### 基本信息

- **项目名**: Claude Code CLI 2.1.88
- **类型**: AI-Powered Terminal IDE / CLI Tool
- **语言**: TypeScript + React (Ink UI Framework)
- **运行时**: Bun (原始设计)
- **主要功能**: 与 Claude AI 模型交互，执行代码任务，管理工程工作流
- **平台**: macOS, Windows, Linux

### 项目目标

Claude Code 是一个革新性的命令行工具，将 Claude AI 助手深度集成到开发者的工作流中，提供：

- **智能代码编辑与生成**
- **自动化任务执行**（Bash, 工作流）
- **MCP（Model Context Protocol）集成**
- **多代理系统**（Agent Swarms）
- **上下文管理与压缩**
- **任务队列与异步执行**
- **会话管理与远程能力**

### 3.1 QueryEngine - 核心查询引擎 ⭐⭐⭐

**文件**: `src/QueryEngine.ts` (~2000+ 行)

**职责**: 处理用户查询的完整生命周期

```typescript
// 核心流程
export async function queryClaudeWithTools(
  userMessage: string,
  appState: AppState,
  abortController: AbortController
): Promise<QueryResult> {
  // 1. 预处理输入
  const processedInput = await processUserInput(userMessage, context)

  // 2. 构建系统提示
  const systemPrompt = await buildSystemPrompt(appState)

  // 3. 调用 Claude API
  const response = await query(systemPrompt, messages, tools)

  // 4. 处理响应
  while (true) {
    // 迭代：模型生成 tool_use，执行，反馈
    const toolCalls = parseToolUses(response)

    for (const toolCall of toolCalls) {
      const result = await executeTool(toolCall)
      messages.push({ role: 'user', content: result })
    }

    // 继续查询直到完成
    response = await query(systemPrompt, messages, tools)
  }

  return finalResult
}
```

**关键特性**:

1. **流式处理**: 支持流式响应，实时显示模型输出
2. **工具执行**: 自动检测和执行 tool_use 块
3. **上下文管理**: 自动构建和管理对话上下文
4. **错误处理**: 重试失败的 API 调用，归类 API 错误
5. **Token 追踪**: 计算和跟踪 Token 使用量
6. **进度报告**: 向 UI 报告执行进度

**数据流**:

```
用户输入
  ↓
processUserInput (权限检查、输入扩展)
  ↓
buildSystemPrompt (收集上下文、构建提示)
  ↓
query() [API 调用]
  ↓
流式处理响应
  ↓
检测 tool_use
  ├─ 工具执行
  │  ├─ BashTool
  │  ├─ FileEditTool
  │  ├─ AgentTool (递归)
  │  └─ ...
  └─ 结果反馈给模型
  ↓
[重复直到停止]
  ↓
输出最终消息
```

### 3.2.6 MCP 客户端连接和工具集成

**设计逻辑**：MCP 客户端遵循"**延迟初始化，按需连接**"原则：

1. 启动时不连接所有服务器（性能）
2. 用户执行工具时动态建立连接
3. 连接缓存重用（避免重复连接）
4. 自动重连和错误恢复

**实现细节** (src/services/mcp/client.ts)：

```typescript
/**
 * MCPClient - 管理所有 MCP 服务器连接
 */
class MCPClient {
  // 连接缓存：Map<serverId, MCPServerConnection>
  private connections = new Map<string, MCPServerConnection>()

  /**
   * 获取或创建连接
   * 关键优化：缓存连接，避免重复建立
   */
  async ensureConnectedClient(
    serverId: string,
    config: ScopedMcpServerConfig
  ): Promise<Client> {
    // 1. 检查缓存
    if (this.connections.has(serverId)) {
      const cached = this.connections.get(serverId)!
      if (cached.isConnected) {
        return cached.client
      }
    }

    // 2. 建立新连接
    const client = new Client({
      name: 'claude-code',
      version: '2.1.88'
    })

    // 3. 选择传输层
    const transport = this.createTransport(config)

    // 4. 连接
    try {
      await client.connect(transport)
    } catch (error) {
      if (error instanceof UnauthorizedError) {
        // OAuth 令牌过期或无效
        throw new McpAuthError(serverId, 'OAuth token expired')
      }
      throw error
    }

    // 5. 初始化（获取能力）
    const capabilities = await client.initialize()

    // 6. 获取工具列表
    const tools = await this.listTools(client, serverId)

    // 7. 缓存连接
    this.connections.set(serverId, {
      id: serverId,
      name: config.name,
      config,
      client,
      tools,
      isConnected: true,
      retryCount: 0
    })

    return client
  }

  /**
   * 创建传输层（根据配置类型）
   */
  private createTransport(
    config: ScopedMcpServerConfig
  ): Transport {
    switch (config.type) {
      case 'stdio':
        // 子进程通信
        return new StdioClientTransport({
          command: config.command,
          args: config.args,
          env: { ...process.env, ...(config.env || {}) }
        })

      case 'sse':
        // 服务端事件流
        return new SSEClientTransport({
          url: config.url,
          headers: await getMcpServerHeaders(config),
          fetch: createFetchWithOAuth(config.oauth)
        })

      case 'http':
        // HTTP 长连接
        return new StreamableHTTPClientTransport({
          url: config.url,
          headers: await getMcpServerHeaders(config),
          fetch: createFetchWithOAuth(config.oauth)
        })

      case 'ws':
        // WebSocket
        return new WebSocketTransport({
          url: config.url,
          // TLS 和代理配置
          tlsOptions: getWebSocketTLSOptions(),
          ...getProxyConfig()
        })

      case 'sdk':
        // SDK 内置服务器
        return createSDKTransport(config.name)

      default:
        throw new Error(`Unknown transport type: ${config.type}`)
    }
  }

  /**
   * 执行 MCP 工具
   */
  async callMCPTool(
    serverId: string,
    toolName: string,
    input: Record<string, unknown>,
    context: ToolUseContext
  ): Promise<MCPToolResult> {
    const connection = await this.ensureConnectedClient(serverId, ...)

    try {
      // 1. 权限检查
      await context.canUseTool('mcp_' + serverId + '_' + toolName)

      // 2. 执行工具
      const result = await connection.request({
        method: 'tools/call',
        params: { name: toolName, arguments: input }
      })

      // 3. 验证结果大小（截断过大输出）
      if (mcpContentNeedsTruncation(result)) {
        result.content = truncateMcpContentIfNeeded(result.content)
      }

      // 4. 持久化二进制内容（图片、文件等）
      if (result.content && containsBinaryData(result.content)) {
        await persistBinaryContent(result.content)
      }

      return result
    } catch (error) {
      if (error instanceof McpSessionExpiredError) {
        // 会话过期，清除缓存并重试
        this.connections.delete(serverId)
        return this.callMCPTool(serverId, toolName, input, context)
      }

      if (error instanceof McpAuthError) {
        // 认证失败，更新客户端状态
        context.updateMcpServerStatus(serverId, 'needs-auth')
      }

      throw error
    }
  }
}
```

**OAuth 认证流程** (src/services/mcp/auth.ts)：

```typescript
/**
 * OAuth 流程：
 * 1. 检测工具需要 OAuth
 * 2. 检查缓存令牌（Keychain）
 * 3. 如果过期，启动 OAuth 流
 * 4. 保存新令牌
 * 5. 重新执行工具
 */

export async function ensureOAuthToken(
  serverId: string,
  oauthConfig: McpOAuthConfig
): Promise<string> {
  // 1. 检查缓存令牌
  const cached = await loadOAuthTokenFromKeychain(serverId)
  if (cached && !isTokenExpired(cached)) {
    return cached.access_token
  }

  // 2. 启动 OAuth 流
  const { authUrl, state } = await getAuthorizationUrl(
    oauthConfig.authServerMetadataUrl,
    oauthConfig.clientId,
    oauthConfig.callbackPort
  )

  // 3. 打开浏览器让用户授权
  await openBrowserForOAuth(authUrl)

  // 4. 启动本地服务器监听 callback
  const code = await waitForOAuthCallback(
    state,
    oauthConfig.callbackPort
  )

  // 5. 交换 code 为 token
  const tokenResponse = await exchangeCodeForToken(
    oauthConfig.authServerMetadataUrl,
    code,
    oauthConfig.clientId
  )

  // 6. 保存到 Keychain（macOS）或凭证管理器（Windows）
  await saveOAuthTokenToKeychain(serverId, tokenResponse)

  return tokenResponse.access_token
}
```

```typescript
/**
 * 执行单个工具调用
 * 包括权限检查、参数验证、错误处理
 */

export async function executeTool(
  toolUse: ToolUseBlock,
  context: ToolUseContext
): Promise<ToolResult> {
  const { name, id, input } = toolUse

  // 1. 查找工具
  const tool = findToolByName(name, context.allTools)
  if (!tool) {
    return {
      type: 'error',
      error: `Tool not found: ${name}`
    }
  }

  // 2. 权限检查
  const canUse = await context.canUseTool(name)
  if (!canUse.allowed) {
    return {
      type: 'error',
      error: `Tool use denied: ${canUse.reason || 'Permission denied'}`
    }
  }

  // 3. 参数验证（JSON Schema）
  const validation = validateToolInput(input, tool.inputSchema)
  if (!validation.valid) {
    return {
      type: 'error',
      error: `Invalid input: ${validation.error}`
    }
  }

  // 4. 执行工具
  try {
    // 超时保护
    const timeout = 30000  // 30 秒
    const result = await Promise.race([
      tool.invoke(input as Record<string, unknown>, context),
      sleep(timeout).then(() => {
        throw new Error(`Tool execution timeout after ${timeout}ms`)
      })
    ])

    return result
  } catch (error) {
    return {
      type: 'error',
      error: String(error),
      message: error instanceof Error ? error.message : undefined
    }
  }
}
```

```typescript
/**
 * LocalShellTask - 执行本地 Shell 命令
 * 使用 Node.js 的 child_process.spawn
 */

export class LocalShellTask {
  private process: ChildProcess | null = null
  private outputStream: WriteStream | null = null

  /**
   * 启动任务
   */
  async spawn(
    input: {
      command: string
      cwd?: string
      timeout?: number
      env?: Record<string, string>
    },
    context: TaskContext
  ): Promise<TaskHandle> {
    const taskId = generateTaskId('local_bash')

    // 1. 创建输出文件
    const outputPath = getTaskOutputPath(taskId)
    this.outputStream = fs.createWriteStream(outputPath, { flags: 'a' })

    // 2. 启动子进程
    // 关键：使用 shell: true 处理复杂命令（管道、重定向等）
    this.process = spawn('bash', ['-c', input.command], {
      cwd: input.cwd || process.cwd(),
      env: { ...process.env, ...input.env },
      stdio: ['ignore', 'pipe', 'pipe'],  // 捕获 stdout/stderr
      // 关键：不继承终端，否则会阻塞
      detached: false
    })

    // 3. 处理输出流
    let totalOutputSize = 0
    const MAX_OUTPUT_SIZE = 10 * 1024 * 1024  // 10MB 限制

    // 捕获 stdout
    this.process.stdout?.on('data', (chunk: Buffer) => {
      totalOutputSize += chunk.length
      if (totalOutputSize > MAX_OUTPUT_SIZE) {
        this.process?.kill('SIGTERM')
      }
      this.outputStream?.write(chunk)

      // 流式报告进度
      context.onProgress?.({
        type: 'bash',
        status: 'running',
        outputSize: totalOutputSize
      })
    })

    // 捕获 stderr（与 stdout 混合，保留顺序）
    this.process.stderr?.on('data', (chunk: Buffer) => {
      this.outputStream?.write(chunk)
    })

    // 4. 处理完成
    return new Promise((resolve, reject) => {
      const timeout = input.timeout
        ? setTimeout(() => {
            this.process?.kill('SIGTERM')
          }, input.timeout)
        : null

      this.process!.on('exit', (code, signal) => {
        if (timeout) clearTimeout(timeout)

        this.outputStream?.end()

        // 更新任务状态
        const status = code === 0 ? 'completed' : 'failed'
        context.setAppState(prev => ({
          ...prev,
          tasks: new Map(prev.tasks).set(taskId, {
            id: taskId,
            type: 'local_bash',
            status,
            description: input.command,
            exitCode: code ?? undefined,
            outputPath,
            createdAt: Date.now(),
            completedAt: Date.now()
          })
        }))

        resolve({
          taskId,
          exitCode: code ?? 1,
          outputPath,
          cleanup: () => {
            this.process?.kill('SIGKILL')
          }
        })
      })

      this.process!.on('error', (error) => {
        reject(error)
      })
    })
  }

  /**
   * 中止任务
   */
  async kill(taskId: string): Promise<void> {
    if (this.process && !this.process.killed) {
      // 先发送 SIGTERM（温和关闭）
      this.process.kill('SIGTERM')

      // 5 秒后仍未结束则发送 SIGKILL（强制杀死）
      await sleep(5000)
      if (!this.process.killed) {
        this.process.kill('SIGKILL')
      }
    }
  }
}
```

```typescript
/**
 * 自动更新记忆文件
 * 基于对话内容学习
 */
export async function updateMemoryFromConversation(
  messages: Message[],
  appState: AppState
): Promise<void> {
  // 1. 分析对话中的学习点
  const learnings = extractLearningsFromMessages(messages)

  if (learnings.length === 0) return

  // 2. 获取现有记忆
  let memory = await loadMemoryPrompt()

  // 3. 合并新学习
  // 关键：避免重复、保持结构化
  for (const learning of learnings) {
    if (!memory.includes(learning.content)) {
      memory += `\n\n## ${learning.title}\n${learning.content}`
    }
  }

  // 4. 限制记忆大小（防止无限增长）
  if (memory.length > MAX_MEMORY_SIZE) {
    memory = compressMemory(memory)
  }

  // 5. 保存
  const projectMemoryPath = join(cwd(), '.claude', 'memory.md')
  await writeFile(projectMemoryPath, memory)
}

/**
 * 记忆压缩：删除过期或低优先级条目
 */
function compressMemory(memory: string): string {
  const sections = memory.split('\n## ')

  // 按最后访问时间排序，删除较早的条目
  const compressed = sections
    .sort((a, b) => {
      const timeA = extractTimestamp(a)
      const timeB = extractTimestamp(b)
      return timeB - timeA
    })
    .slice(0, MAX_MEMORY_SECTIONS)
    .join('\n## ')

  return compressed
}
```

```typescript
  return create<AppStateStore>((set) => ({

    setState: (updater) => {
      set((prev) => {
        // updater 可以是对象或函数
        const newState =
          typeof updater === 'function' ? updater(prev) : updater

        // 通知观察者
        onChangeAppState?.({ newState, oldState: prev })

        // 持久化关键状态
        if (onChangeAppState) {
          persistAppState(newState)
        }

        return newState
      })
    },
    subscribe: (selector, callback) => {
      // 选择器订阅（仅当选定值变化时触发）
      let prevSelected = selector(getState())

      return set((state) => {
        const newSelected = selector(state)
        if (!Object.is(prevSelected, newSelected)) {
          callback(newSelected)
        }
        prevSelected = newSelected
        return state
      })
    }
  }))
}
```

#### 3.6.3 状态持久化

**文件缓存** (src/utils/fileStateCache.ts)：

```typescript
/**
 * FileStateCache 的核心价值：
 * 1. 避免重复读取文件
 * 2. 追踪文件内容变化（检测修改）
 * 3. LRU 自动清理，内存有界
 */

export class FileStateCache {
  private cache: LRUCache<string, FileState>

  constructor(maxEntries: number = 100, maxBytes: number = 25 * 1024 * 1024) {
    this.cache = new LRUCache({
      max: maxEntries,
      maxSize: maxBytes,
      sizeCalculation: value => Buffer.byteLength(value.content)
    })
  }

  /**
   * 获取或加载文件
   */
  async getOrLoad(
    filePath: string,
    offset?: number,
    limit?: number
  ): Promise<FileState | null> {
    const normalized = normalize(filePath)

    // 1. 检查缓存
    const cached = this.cache.get(normalized)
    if (cached && cached.offset === offset && cached.limit === limit) {
      return cached
    }

    // 2. 从磁盘读取
    try {
      const content = await readFile(filePath, 'utf-8')

      // 3. 应用偏移和限制
      let finalContent = content
      if (offset !== undefined && limit !== undefined) {
        finalContent = content.substring(offset, offset + limit)
      }

      // 4. 缓存
      const state: FileState = {
        content: finalContent,
        timestamp: Date.now(),
        offset,
        limit
      }

      this.cache.set(normalized, state)
      return state
    } catch (error) {
      return null
    }
  }

  /**
   * 检查文件是否被修改
   */
  hasChanged(filePath: string, newContent: string): boolean {
    const cached = this.cache.get(normalize(filePath))
    if (!cached) return true  // 新文件

    return cached.content !== newContent
  }

    this.cache.delete(normalize(filePath))
```

```typescript
/**
 * FileEditTool - 编辑文件
 * 关键特性：
 * 1. 精确行匹配
 * 2. 原子操作（全部成功或全部失败）
 * 3. 备份原文件

export const FileEditTool: Tool = {
  name: 'edit',
  description: 'Edit a file with precise line-based replacement',

  async invoke(input, context) {
    const {
      file_path,
      edits  // 数组：{ old_string, new_string }
    } = input

    try {
      // 1. 权限检查
      const allowed = await context.canUseTool('edit')
      if (!allowed) {
        return { type: 'error', error: 'File edit denied' }
      }

      // 2. 检查文件是否在缓存中
      // 关键：必须先读取文件才能编辑（模型需要看到当前内容）
      const cached = context.readFileCache.get(String(file_path))
      if (!cached) {
        return {
          type: 'error',
          error: 'File must be read before editing'
        }
      }

      // 3. 应用所有编辑
      let content = cached.content
      for (const edit of edits as Array<{ old_string: string; new_string: string }>) {
        if (!content.includes(edit.old_string)) {
          return {
            type: 'error',
            error: `Pattern not found: ${edit.old_string.slice(0, 50)}...`
          }
        }
        content = content.replace(edit.old_string, edit.new_string)
      }

      // 4. 备份原文件
      const backupPath = `${file_path}.bak`
      await writeFile(backupPath, cached.content)

      // 5. 写入新内容
      await writeFile(String(file_path), content)

      // 6. 清除缓存
      context.readFileCache.invalidate(String(file_path))

      return {
        type: 'text',
        text: `Edited ${file_path} (backup saved to ${backupPath})`
      }
    } catch (error) {
      return { type: 'error', error: String(error) }
    }
  }
}
```

### 3.9 Query Engine 深入分析

Query Engine (src/QueryEngine.ts, src/query.ts) 是 Claude Code 的核心，负责处理用户查询的完整生命周期。

#### 3.9.1 查询循环核心流程

**设计逻辑**：
Query Engine 实现了一个**迭代生成和工具执行循环**，利用 Claude API 的工具调用能力自动化任务。核心思想：
1. 调用模型生成响应（可能包含工具调用）
2. 执行所有工具调用
3. 反馈工具结果给模型
4. 重复直到模型停止（end_turn）或达到最大迭代次数

**实现细节** (src/query.ts 第123-200行)：

```typescript
/**
 * 流式查询执行的骨架
 * 支持多种中断条件和错误恢复
 */

export async function* queryInternal(
  params: QueryParams
): AsyncGenerator<StreamEvent | Message, Terminal | Continue, undefined> {

  const {
    messages,
    systemPrompt,
    canUseTool,
    toolUseContext
  } = params

  let iteration = 0
  const maxTurns = params.maxTurns ?? 20
  const messageHistory = [...messages]

  while (iteration < maxTurns) {
    iteration++

    // 1. 调用 Claude API（流式）
    // 关键优化：使用 stream=true 实现实时输出
    const apiCall = await callClaudeAPI({
      system: systemPrompt,
      messages: normalizeMessagesForAPI(messageHistory),
      tools: availableTools,
      stream: true,
      // 预算管理（token/cost）
      maxOutputTokens: calculateMaxOutputTokens(),
      taskBudget: params.taskBudget
    })

    // 2. 流式处理响应
    // 关键特性：实时 yield 文本片段给 UI
    let assistantMessage: AssistantMessage = {
      uuid: generateUUID(),
      message: { role: 'assistant', content: [] },
      stopReason: 'end_turn'
    }

    let hasToolUse = false

    for await (const event of apiCall.stream) {
      // 事件类型检测
      if (event.type === 'content_block_delta') {
        if (event.delta.type === 'text_delta') {
          // 实时流式输出文本
          yield {
            type: 'stream_event',
            event: {
              type: 'content_block_delta',
              delta: { type: 'text_delta', text: event.delta.text }
            }
          }
          assistantMessage.message.content.push({
            type: 'text',
            text: event.delta.text
          })
        }
      }

      // 工具调用块检测
      if (event.type === 'content_block_stop' &&
          event.content_block.type === 'tool_use') {
        hasToolUse = true
        assistantMessage.message.content.push(event.content_block)
        yield {
          type: 'tool_use_detected',
          toolName: event.content_block.name
        }
      }

      // Token 使用情况
      if (event.type === 'message_delta') {
        assistantMessage.stopReason = event.message.stop_reason
      }
    }

    // 3. 添加助手消息到历史
    messageHistory.push(assistantMessage)
    yield assistantMessage  // 发送给 UI

    // 4. 如果没有工具调用，流程结束
    if (!hasToolUse || assistantMessage.stopReason === 'end_turn') {
      return { type: 'terminal', reason: 'model_stopped' }
    }

    // 5. 执行所有工具调用
    // 关键优化：并行执行多个工具调用
    const toolUses = assistantMessage.message.content.filter(
      c => c.type === 'tool_use'
    )

    const toolResults = await runTools({
      toolUses,
      context: toolUseContext,
      canUseTool,
      iteration
    })

    // 6. 创建工具结果消息
    messageHistory.push({
      role: 'user',
      content: toolResults
    })

    // 继续循环...
  }

  // 达到最大迭代次数
  return { type: 'terminal', reason: 'max_turns_exceeded' }
}
```

**关键机制**：

1. **流式处理**（src/query.ts）：
   - 使用 `AsyncGenerator` 实现流式处理
   - 每个文本块都被立即 yield 到 UI
   - 提供实时反馈，改善用户体验

2. **工具执行隔离**（src/services/tools/toolOrchestration.ts）：

   - 错误处理：捕获工具执行异常
   - 结果格式化：所有输出转为标准格式

3. **预算管理**：
   - Token 预算追踪（src/utils/tokens.js）

```typescript
/**
```

```typescript
/**
 * API 错误分类和恢复策略
 */

export async function queryWithRetry(
  params: QueryParams
): Promise<QueryResult> {
  let lastError: Error | null = null
  const maxRetries = 3
  let retryCount = 0

  while (retryCount < maxRetries) {
    try {
      return await query(params)
    } catch (error) {
      lastError = error as Error
      const category = categorizeRetryableAPIError(error)

      switch (category) {
        case 'rate_limit':
          // 指数退避
          await sleep(Math.pow(2, retryCount) * 1000)
          retryCount++
          break

        case 'server_error':
          // 重试一次
          if (retryCount < 1) {
            await sleep(500)
            retryCount++
          } else {
            throw error
          }
          break

        case 'auth_error':
          // 刷新 token 后重试
          await refreshAuthToken()
          retryCount++
          break

        default:
          // 其他错误直接抛出
          throw error
      }
    }
  }

  throw lastError || new Error('Query failed after retries')
}
```

```typescript
async function coordinatorLoop(
  coordinatorDefinition: AgentDefinition,
  initialTask: string
): Promise<CoordinatorResult> {

  const coordinator = spawnAgent(coordinatorDefinition)
  const workers = new Map<string, WorkerAgent>()

  // 1. 协调器分析任务
  const taskPlan = await coordinator.analyze(initialTask)

  // 2. 分配子任务给工作代理

    // 异步启动工作代理
    worker.executeAsync(subtask.description)

  )

  const finalAnswer = await coordinator.synthesize(results)

  return finalAnswer
```

## 核心实现模式和最佳实践

### 1. 迭代和工具执行循环 (Query Loop)

**核心原理**：Claude Code 实现了一个**模型驱动的自动化执行引擎**：

```
1. 模型生成响应
   ↓ [可能包含 tool_use 块]
2. 检测工具调用
   ↓
3. 执行所有工具
   ↓
4. 反馈结果给模型
   ↓ [模型继续推理]
5. 重复直到 end_turn
```

**关键特性**：
- **并行工具执行**：多个工具调用同时进行
- **流式输出**：文本实时显示，不等待完成
- **错误恢复**：单个工具失败不影响其他工具
- **预算管理**：Token 和成本追踪

### 2. 权限系统 (分层检查)

**设计原则**：信任但可验证

```
权限检查流程：
  1. 全局权限模式（default/bypass/ask/auto）
  2. 工具级黑名单（deny rules）
  3. 工具级白名单（allow rules）
  4. 特定操作检查（如文件路径）
  5. 用户确认（如需要）
  6. 决策记录（审计日志）
```

### 3. 状态管理 (单一真实源)

**核心原则**：所有 UI 状态来自 AppState

```typescript
// ✅ 正确方式：通过 setState 更新
setAppState(prev => ({
  ...prev,
  messages: [...prev.messages, newMessage]
}))

// 使用选择器避免不必要重新渲染
const messages = useAppState(s => s.messages)
```

### 4. 文件操作的安全性

**三步验证流程**：

```
1. Read First Rule
   ↓ 必须先读取文件，模型才能修改

2. Cache Check
   ↓ 检查文件是否在缓存中（防止盲编辑）

3. Atomic Write
   ↓ 编辑全部成功或全部失败，带备份
```

### 5. MCP 集成的优化

**延迟初始化策略**：

```
启动时：只加载配置
  ↓
第一次使用：建立连接
  ↓
连接缓存：重用连接
  ↓
错误时：清除缓存，重新连接
```

### 6. 上下文压缩策略

**优先级**（从严厉到宽松）：

```
1. 文件级别：只读取必要部分
2. 工具输出级别：截断过长输出（5KB）
3. 消息级别：删除早期消息，保留最新
4. Token 级别：使用提示缓存（ephemeral）
```

### 7. 错误处理和恢复

**分类处理**：

```typescript
API 错误
  ├─ 速率限制 → 指数退避
  ├─ 服务器错误 → 重试一次
  ├─ 认证错误 → 刷新 token 后重试
  └─ 其他错误 → 直接抛出

工具执行错误
  ├─ 权限拒绝 → 返回错误消息
  ├─ 参数验证失败 → 返回验证错误
  ├─ 超时 → 自动中止（30秒）
  └─ 其他异常 → 返回错误，继续循环
```

## 关键设计决策详解

### 为什么使用 AsyncGenerator 处理流式响应？

**答**：
1. **资源高效** - 流式处理避免一次性加载全部内容
2. **用户体验** - 实时反馈，不需要等待完成
3. **可取消** - 用户可以中断长操作
4. **错误恢复** - 部分失败不影响已处理部分

### 为什么分离 FileStateCache 和权限检查？

**答**：
1. **单一职责** - 缓存只管缓存，权限只管权限
2. **可测试性** - 每个模块独立测试
3. **性能** - 缓存命中快速返回，权限检查不触及磁盘
4. **安全** - 权限规则容易审计和修改

### 为什么使用 Zustand 而不是 Redux？

**答**：
1. **最小化** - 代码量少，学习曲线平缓
2. **不需要中间件** - 对于 CLI 应用够用
3. **订阅模式** - 支持选择器，避免不必要重新渲染
4. **异步友好** - async/await 原生支持

### 为什么 MCP 支持多种传输方式？

**答**：
1. **灵活性** - 不同场景不同方式（stdio, HTTP, WebSocket）
2. **隔离** - 子进程 stdio 完全隔离，更安全
3. **互操作** - 与其他工具共享 MCP 服务器
4. **扩展** - 简化添加新传输方式
